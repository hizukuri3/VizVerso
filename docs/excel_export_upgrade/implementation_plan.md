# Excel出力機能のアップグレード計画

Excel出力機能を、UI側の進化（パラメータ詳細、依存関係の可視化等）に合わせて強化し、データ加工がしやすい「縦持ち（Tidy data）」形式に全面的に刷新します。

## ユーザーレビューが必要な事項

- **ワークシート別詳細シートの維持**: ユーザー様のご要望通り、各ワークシートごとの詳細シート（レイヤーやマーク情報）は維持します。
- **「要素」の粒度について**: ここで言う要素とは、**「1つのフィールドの割り当て（列、行、フィルター、エンコーディング等）」**を指します。例えば、1つのワークシートの「行」に3つのフィールドが配置されている場合、これまでは1つのセルにカンマ区切りで入っていましたが、アップグレード後は3行（3要素）として出力します。

## 提案される変更点

### 1. 多言語対応の追加 (`src/locales/ja.json`, `src/locales/en.json`)

新しいExcelシート名やカラムヘッダーのための翻訳キーを追加します。

### 2. Excel出力ロジックの刷新 (`src/utils/excelExporter.ts`)

以下の新しいシート構成で実装を書き換えます。

#### A. ダッシュボード構成 (`t('excel.sheet_dashboard')`)

- 縦持ちに変更。
- カラム: `[ダッシュボード名, ワークシート名]`

#### B. ワークシート・シェルフ (`t('excel.sheet_worksheet_shelf')`) [NEW]

- 全てのワークシートの「列」「行」「フィルター」を縦持ちでリスト化。
- カラム: `[ワークシート名, データソース, シェルフ名 (列/行/フィルター), フィールド名]`

#### C. ワークシート個別の詳細シート [MAINTAINED]

- 従来通り、ワークシートごとにシートを作成します。
- 内部のデータは縦持ち（1エンコーディング＝1行）を徹底します。
- カラム: `[レイヤー/軸, マーク種類, 役割 (色/サイズ/ラベル/詳細/ツールヒント), 集計, フィールド名]`

#### D. パラメータ一覧 (`t('excel.sheet_parameters')`) [NEW]

- ワークブック内の全パラメータの設定値をリスト化。
- カラム: `[パラメータ名, データ型, 許容値 (すべて/リスト/範囲), 設定詳細]`

#### E. フィールド一覧 & 計算式 (`t('excel.sheet_fields')`)

- 標準フィールドと計算フィールドを統合（または並記）。
- カラム: `[データソース, フィールド名, データ型, ロール, 計算式]`

#### F. 依存関係一覧 (`t('excel.sheet_dependencies')`) [NEW]

- ワークシートがどのフィールドを使用しているかをリスト化。
- カラム: `[ワークシート名, 使用フィールド名]`

## 検証計画

### 自動テスト / ブラウザテスト

- `npm run dev` でアプリを起動。
- サンプルファイルをアップロード。
- 「Excel 出力」を実行し、生成されたファイルを開いて以下の点を確認する。
  - 各シートが縦持ち形式になっていること（カンマ区切りのセルがないこと）。
  - パラメータの情報が正しく出力されていること。
  - 日本語/英語の切り替えが正しく反映されていること。

### 手動確認

- Excelの「データ」→「フィルター」を使用して、特定のワークシートやフィールドで絞り込みができることを確認。
