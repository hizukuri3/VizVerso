# 実装計画 (Implementation Plan) - Marks N 表記の解決

## 概要
Tableau ワークブックの二重軸や複数メジャー使用時、UI や Excel 出力において「Marks 2」「Marks 3」といった内部的な識別子が表示されてしまう問題を解決します。

## 解決策
Tableau の XML における `pane` 要素の `@_y-axis-name` または `@_x-axis-name` 属性を利用し、どの軸（フィールド）に対応するペインかを特定し、その表示名を使用するように変更します。

## 変更内容
1.  **xmlParser.ts**: `WorksheetPane` 型に `yAxisName`, `xAxisName` を追加し、解析時にこれらを保持するように修正。
2.  **DetailView.tsx**: ペインのタイトルを表示する際、軸名が存在すればそのフィールド表示名を優先して使用するように修正。
3.  **excelExporter.ts**: Excel 出力の「レイヤー/軸」列において、同様にフィールド表示名を優先するように修正。

## 完了条件
- 二重軸のシートで「Marks 2」ではなくフィールド名が表示されること。
- Excel 出力でも正しい名称が表示されること。
- 「全体」と「すべて」の表記が「すべて」に統一されていること。
