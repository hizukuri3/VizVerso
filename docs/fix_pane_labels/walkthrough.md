# 修正内容の確認 (Walkthrough)

二重軸や複数メジャーを持つワークシートにおいて、内部的な識別子（Marks 2, Marks 3 など）が表示されていた問題を解決しました。

## 実施した変更

### 1. XML 解析の強化 (xmlParser.ts)
- `panes/pane` 要素から `@_y-axis-name` および `@_x-axis-name` 属性を抽出するようにしました。
- これらの属性は、そのペインがどの軸（メジャー）に対応しているかを示す参照名です。

### 2. UI 表示の改善 (DetailView.tsx)
- ペインのラベルを決定する際、まず `yAxisName` または `xAxisName` が存在するか確認します。
- 存在する場合、その参照名から `getFieldInfo` を通じて正しいフィールド表示名（例：`合計(SelectedSort)`）を取得して表示するようにしました。
- これにより、二重軸の際も「Marks 2」ではなくフィールド名が表示されます。

### 3. Excel 出力の改善 (excelExporter.ts)
- UI と同様のロジックを適用し、Excel の「レイヤー/軸」列に内部 ID ではなく正しい表示名が出力されるようにしました。
- また、共通のペインについては一貫して「すべて」と表記するように統一しました。

## 検証結果
- `npm run build` により、型エラーがないことを確認しました。
- 二重軸（BackGround シートなど）において、軸に対応するフィールド名が正しく解決されるロジックになっています。
