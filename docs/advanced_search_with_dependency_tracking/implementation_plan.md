# 実装計画: 依存関係を考慮した高度な検索機能

## 目的
計算フィールドの依存関係（どのフィールドがどの計算式で使われているか）を解析し、単なる名称一致を超えた「芋づる式」の検索を可能にすることで、データの繋がりを直感的に把握できるナビゲーションを構築する。

## 提案内容

### 1. 依存関係解析エンジン (`useSearch.ts`)
- **インデックス化**: ワークブック読み込み時に、計算式 (`formula`) をパースし、フィールド間の参照関係（親子関係）およびワークシートでの使用箇所をグラフ構造としてメモリ上に構築。
- **カスケード検索ロジック**: 
  - 検索語がフィールド名にヒットした場合、そのフィールドを参照している「親」の計算フィールドも検索結果に含める。
  - 同時に、そのフィールドを使用しているワークシートもヒットさせる。

### 2. UI/UX の刷新
- **ヘッダー検索バー**: 既存の静的な表示を撤廃し、クリアボタン付きのモダンな入力フォームを配置。
- **検索結果リスト (`SearchResultsList.tsx`)**:
  - ヒット理由の明示（「直接一致」「計算式内ヒット」「依存先によるヒット」などのバッジ表示）。
  - アイテムの種類（シート/データソース/フィールド）に応じたアイコン表示。
  - 遷移の最適化：フィールドクリック時はその定義元（データソース）へ、シートクリック時はその詳細へ遷移。

### 3. パフォーマンス最適化
- `useMemo` を活用し、ドキュメントが更新されない限りインデックスの再構築を抑制。
- 検索実行時も計算済みのグラフをスキャンするのみとし、低遅延なレスポンスを実現。

## 変更ファイル
- `src/hooks/useSearch.ts`: 検索ロジックと依存グラフ管理
- `src/components/SearchResultsList.tsx`: 検索結果表示用コンポーネント
- `src/App.tsx`: ヘッダーへの統合とナビゲーション制御
- `src/locales/ja.json`, `en.json`: 検索関連の文言追加
