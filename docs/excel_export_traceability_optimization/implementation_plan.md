# 実装計画：Excelエクスポートのトレーサビリティ向上

Tableauワークブック内のフィールド依存関係を正確に解析し、Excelレポートの品質を向上させる。

## 課題

- 計算フィールドのデータソースが、XML上の配置場所に引きずられて不正確になる。
- 物理名（Calculation\_...）が表示され、可読性が低い。
- 定数計算に不要なデータソース名が表示される。
- 標準フィールド（ID等）が解析から漏れる。

## 解決策

1. **計算式解析エンジンの実装**
   - 正規表現を用いて、数式内の `[Field]` および `[DS].[Field]` 参照を抽出。
   - 再帰的な解決により、多階層の依存関係を網羅。
2. **メタデータ抽出の拡張**
   - `<metadata-record>` から標準的なデータベース項目をすべて捕捉。
3. **キャプション置換ロジック**
   - 数式内の物理名を、表示名（Caption）に動的に書き換えて出力。
4. **表示ルールの洗練**
   - 依存先がない場合は空欄、パラメータ依存は「パラメータ」と明示。

## 検証項目

- クロスデータソース計算の紐付け。
- リレーションシップ環境でのテーブル特定。
- 物理名が含まれないクリーンな数式の出力。
