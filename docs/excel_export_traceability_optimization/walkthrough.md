# Tableau Workbook Analyzer: Excel Export Traceability Optimization

Tableau Workbook の Excel エクスポート機能において、データソースの紐付け精度と計算式の可読性を極限まで高めるための最適化を実施しました。

## 実施した主な改善

### 1. 計算式ベースの依存関係解決エンジン

- **実態優先の解析**: XML上の物理的な定義場所ではなく、フィールドの計算式（formula）をパースして依存先を特定するロジックを実装しました。
- **クロスデータソース対応**: `[Datasource].[Field]` 形式の参照を正確に検出し、複数データソースにまたがる依存関係もカンマ区切りで表示可能にしました。
- **多階層解決**: 計算フィールドが別の計算フィールドを参照している場合も、最大5パスの解析により最終的なデータソースまで辿り着けるようにしました。

### 2. 標準フィールドの完全捕捉

- **メタデータスキャンの強化**: XML内の `<metadata-record>` セクションを解析対象に追加しました。これにより、画面上で別名がついていない `ID` などの標準的なデータベース項目も正確に認識し、依存関係の解決に使用できるようになりました。

### 3. 表示名（Caption）への自動置換

- **人間が読める数式**: Excelに出力されるすべての計算式において、`[Calculation_12345...]` のような内部物理名を、ユーザーが設定した表示名（Caption）に自動置換するエンジンを搭載しました。

### 4. 特殊項目の表示ルール最適化

- **定数計算の空欄化**: `MAKEPOINT(0,0)` などの外部データに依存しない定数計算や、システム生成のマップレイヤー項目などは、データソース列をあえて「空欄」にするルールを適用しました。
- **パラメータの識別**: パラメータへの依存を明確に「パラメータ」として表示するようにし、データソースとの混同を防ぎました。

## 検証結果

- `isVisible`: 数式内の `pName` (パラメータ) と `Brewery Name` (データソース) の両方が正しく抽出されていることを確認。
- `isSort1`: パラメータのみに依存している場合、データソース列が「パラメータ」のみ（または空欄）になっていることを確認。
- `loc.line`: `ID` への依存を介して、正しい親データソース（San Diego Brewery Ratings...）が紐付いていることを確認。
- `mp.origin`: `MAKEPOINT(0,0)` という定数計算に対し、データソース列が正しく空欄になっていることを確認。
