## 2. 詳細詳細

### 2.1 セキュリティ & 堅牢性

- **XSS**: `dangerouslySetInnerHTML` の使用はなく、Reactの標準的なエスケープにより保護されています。合格。
- **インジェクション攻撃**:
  - URLパラメータ経由の攻撃に対しては、内部的な文字列一致による検索のため安全です。
  - **[修正済み]** `FormulaHighlighter` 内の正規表現生成において、検索クエリのエスケープ処理を実装しました。これにより `[` などの特殊文字入力時も正常に動作します。
- **サードパーティ・ライセンス**: 全ての新規ライブラリ（`fast-xml-parser`, `xlsx` 等）が商用利用可能なライセンス（MIT/Apache等）であることを確認しました。合格。

### 2.2 パフォーマンス & スケーラビリティ

- **計算負荷**:
  - 依存関係の探索（BFS）は訪問済みセットと深さ制限（5）により適切に制御されています。
  - **[修正済み]** 検索入力（`onChange`）に対して300msのデバウンスを適用しました。これにより一文字ごとの不要な再計算が抑制され、パフォーマンスが改善しました。
- **メモリリーク**:
  - SideDrawerの開閉に伴う `body.overflow` の制御やイベントリスナーのクリーンアップは適切に行われています。合格。

### 2.3 ユーザー体験 (UX) の連続性

- **文脈保持**:
  - `scrollIntoView` による自動スクロールが正常に機能し、ビューポート外の項目も強調表示されます。合格。
  - サブペインを閉じてもメイン画面のスクロール位置は維持されています。合格。
- **エッジケース**:
  - 正規表現の特殊文字入力時の挙動はエスケープ処理により安全に制御されています。

### 2.4 リーガル & コンプライアンス

- **100%クライアントサイド**: すべてのパース・解析処理がブラウザ内で完結し、外部サーバーへのデータ送信は一切行われないことをコードレベルで確認しました。
- **IP保護**: テレメトリ（GA/Sentry等）は導入されておらず、ユーザーの計算ロジック（知的財産）が外部に漏洩する経路は存在しません。
- **ライセンス**: 使用している全ライブラリ（React, JSZip, xlsx等）が商用利用可能な許諾型ライセンス（MIT/Apache等）であることを確認しました。

## 3. 最終修正アクション

以下の修正を実施した上でリリース判定を「合格」とします。

1.  `FormulaHighlighter.tsx` での `searchQuery` エスケープ処理。
2.  `App.tsx` での検索クエリ入力に対するデバウンス処理の追加。
