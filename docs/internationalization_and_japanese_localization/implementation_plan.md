# 実装計画: 国際化（i18n）および日本語化

## 目的
UI上のハードコードされたテキストを排除し、言語リソースファイル（JSON）に移行することで、文言の統一感（自然な日本語）を持たせるとともに、将来的な多言語展開を容易にする。

## 提案内容

### 1. i18n 管理システムの導入
- `src/locales/` 配下に言語ごとの JSON ファイルを配置。
- `src/utils/i18n.ts` を中心とした、型定義によるミス防止機能付きの翻訳ユーティリティ。

### 2. 対象範囲
- 全ての UI コンポーネント内のラベル、メッセージ、ツールチップ。
- Excel エクスポート時のシート名、ヘッダー。
- Tableau 特有の用語（マークタイプ、集計関数名）の変換。

### 3. リファクタリング戦略
- 文字列を直接参照せず、`t('path.to.key')` 形式で呼び出す。
- 動的な数値や名称が含まれる場合は `t('key', { name: val })` 形式で補間。
- 集計関数やマークタイプは、専用の変換関数 (`tAgg`, `tMark`) を通して解決。

## 変更ファイル
- `src/locales/ja.json`: 日本語リソース
- `src/locales/en.json`: 英語リソース
- `src/utils/i18n.ts`: 翻訳エンジン
- `src/utils/excelExporter.ts`: Excel 出力ロジック
- 各 React コンポーネント (`App.tsx`, `DetailView.tsx` 等)

## 確認方法
- ブラウザ上で全ての表示が日本語になっていることを目視確認。
- 言語設定を切り替えた際に（将来的な機能）、文言が適切に追従することを確認。
- Excel を出力し、メタデータが日本語化されていることを確認。
