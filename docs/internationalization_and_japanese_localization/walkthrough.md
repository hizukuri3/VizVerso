# 国際化（i18n）および日本語化対応

アプリケーションの全ての表示テキストを外部リソースファイルに移行し、自然な日本語への統一および将来的な多言語展開が可能な構造へとリファクタリングを完了しました。

## 実施内容

### 1. 国際化基盤の構築

- `src/locales/ja.json` および `en.json` を作成し、文言を一元管理。
- `src/utils/i18n.ts` に型安全な翻訳関数 `t()` を実装。
- ネストされたキーパス（例: `nav.dashboards`）や動的パラメータ（例: `{{count}}`）に対応。

### 2. UIコンポーネントの完全ローカライズ

以下のコンポーネント内のハードコードされた文字列を排除し、`t()` 関数による参照に置き換えました：

- `App.tsx` (ブランディング、エラー、メッセージ)
- `Sidebar.tsx` (ナビゲーション)
- `DetailView.tsx` (詳細表示、集計関数名)
- `DragDropZone.tsx` (ファイルアップロード)
- `AboutModal.tsx` (アプリ情報、免責事項)
- `Breadcrumbs.tsx` (パンくずリスト)
- `Pill.tsx` (フィールド表示、計算式ハイライト)

### 3. Excel エクスポート機能の多言語対応

- `excelExporter.ts` を修正し、出力されるシート名、カラムヘッダー、集計値を言語設定に応じて動的に切り替えるように対応しました。

### 4. 特殊ロジックの国際化

- **計算式ハイライト**: `Pill.tsx` 内の正規表現を修正し、各言語の「パラメーター」データソース名を正しく識別できるようにしました。
- **マークタイプ**: `tMark()` 関数により、Tableau 特有のマークタイプ（Bar, Area等）を自然な日本語（棒, エリア等）に変換して表示します。

## 検証結果

- [x] UI上の全てのテキストが日本語で表示されること。
- [x] Excel出力のファイル名や内容が日本語化されていること。
- [x] 言語切り替え時に即座に反映される基盤が整っていること（現在は `ja` 固定、`setLanguage` で切り替え可能）。
- [x] 計算式のハイライトや物理名の表示が崩れていないこと。

## 今後のメンテナンス

新しい文言を追加する場合は、`src/locales/ja.json` および `en.json` にキーを追加し、コード内から `t('new.key')` で呼び出してください。型定義により、存在しないキーを指定した場合はビルドエラーが発生するため、安全に拡張可能です。
