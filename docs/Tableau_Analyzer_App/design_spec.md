# Tableau Workbook Analyzer デザイン設計書 (Design Specification)

## 1. 画面構成（レイアウト）

本アプリケーションは、ブラウザ幅を広く使うフルスクリーンのSPA (Single Page Application) として構成します。

### 1.1 全体レイアウト
| エリア | 配置 | 役割・機能 |
| :--- | :--- | :--- |
| **Header** | 上部 (固定) | アプリケーションのロゴ、タイトル、`.twbx` ファイルのアップロード/ドラッグ＆ドロップ領域 |
| **Sidebar** | 左側 (幅固定/可変) | ダッシュボード、シート、データソースの階層リスト (ツリービュー)。運用者向けのメタデータ（未使用フィールド一覧など）のナビゲーション |
| **Main Canvas** | 中央 (大部分) | **React Flow** を用いた依存関係のネットワーク図（Tableau Prep風のフローグラフ） |
| **Detail Panel** | 右側 (またはフローティング) | キャンバス上で選択したノード（シートや計算フィールド）の詳細情報。特に計算ロジックの中身をシンタックスハイライト付きで表示する |

## 2. カラースキーム (Tableau 10 Color Palette)

Tableauに親和性のある、識別しやすく美しいカラーパレットを採用します。

*   **Primary (青)**: `#1f77b4` (選択状態やアクションボタン)
*   **Secondary (オレンジ)**: `#ff7f0e` (警告や特定ノードのハイライト)
*   **Success (緑)**: `#2ca02c` (正常な抽出データソースなど)
*   **Danger (赤)**: `#d62728` (エラー、切断された依存関係など)
*   **Neutral (紫・茶・ピンク等)**: シートやフィールドの種別ごとの色分けに使用
*   **Background**: `#f8f9fa` (全体背景), `#ffffff` (パネルやキャンバス背景)
*   **Text**: `#333333` (基本テキスト), `#6c757d` (サブテキスト)

## 3. UIコンポーネント構成

Reactで構築する主要コンポーネントの階層構造です。

```text
App
 ├── Header
 │    └── FileUploader (ドラッグ＆ドロップ対応)
 ├── Sidebar
 │    ├── OverviewTab (シート一覧ツリー)
 │    └── InsightsTab (未使用フィールド等の運用者向けメタデータ)
 ├── Workspace
 │    ├── NetworkGraph (React Flow キャンバス)
 │    │    ├── CustomNode (アイコン付きノード)
 │    │    └── CustomEdge (Prep風の曲線エッジ)
 │    └── Toolbar (ズーム、レイアウト自動整理ボタン)
 └── DetailPanel
      ├── FieldProperties (フィールドの型や所属)
      └── CalculationViewer (計算式のコード表示)
```

## 4. ネットワーク図（React Flow）の設計

*   **ノードのデザイン**: 
    *   角丸の四角形（白背景、薄いシャドウ）。
    *   ノードの左端に、色付きのアイコン（データソース=青い円柱、シート=緑の表、フィールド=#アイコン等）を配置。
*   **エッジ（接続線）のデザイン**:
    *   Tableau Prep のように、滑らかな曲線（Bezier）を使用。
    *   データが流れる方向を示すアニメーション（点線の移動など）はオプションとして検討。
