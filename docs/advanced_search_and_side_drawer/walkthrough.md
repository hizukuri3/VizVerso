# 実装完了: 高度な検索機能とサイドペイン詳細表示

## 変更内容
計算フィールドの依存関係（参照元・参照先）を考慮した検索機能と、詳細情報を即座に確認できるサイドドロワーを実装しました。

### 1. 検索機能の強化
- **高度な依存関係解析**: `useDependencyIndex` フックを新規作成し、計算式内のフィールド参照をインデックス化しました。
- **検索範囲の拡大**: フィールド名だけでなく、計算式の内容や依存関係に基づいたヒットを可能にしました。
- **検索結果リスト**: ヒット理由（直接一致、計算式内、依存関係）をバッジで表示し、遷移先の定義元（データソース）が明確にわかるようにしました。

### 2. URLパラメータによるコンテキスト維持
- **自動遷移ロジック**: 検索結果をクリックすると、`?type=datasource&id=DS_NAME&field=FIELD_NAME` のように URL パラメータを付与して遷移します。
- **自動ペイン展開**: 遷移先画面の `useEffect` でパラメータを検知し、対象のフィールド詳細を即座にサイドドロワーで表示します。

### 3. サイドドロワー (Side Drawer)
- **シンタックスハイライト**: `FormulaHighlighter` コンポーネントを新規作成し、Tableau 計算式を色分け表示します。
- **検索キーワード強調 & 自動スクロール**: 計算式内の検索キーワードを強調表示し、最初のヒット箇所まで自動でスクロールします。
- **依存関係の可視化**: 「参照元 (Upstream)」と「参照先 (Downstream)」をリスト表示し、クリックすることで詳細情報を切り替え（ドリルダウン）可能です。

### 4. 視覚的フィードバック
- **バッジの強調表示**: サイドドロワーが開いている間、メイン画面上の該当するフィールドバッジ（Pill）を枠線とフラッシュアニメーションで強調表示します。
- **自動スクロール**: メイン画面でも該当するバッジが画面外にある場合、中央に来るように自動スクロールします。

### 5. フィードバックに基づく改善
- **計算式内の物理名置換**: 共有ユーティリティ `formulaFormatter.ts` を作成し、サイドドロワーおよびメイン画面の両方で、計算式内の `[Calculation_...]` などの物理名を表示名（Caption）に置換するようにしました。
- **ハイライト中のポップアップ表示**: ハイライト（アクティブ）状態のバッジでも、マウスホバー時に詳細ポップアップが表示されるように修正しました。
- **領域外クリックによる解除の挙動調整**: メイン画面の空き領域をクリックした際、ハイライト（強調）を解除するロジックを実装しました。ドロワーが開いている間だけでなく、ドロワーを閉じた後もハイライトが残っている状態で領域外をクリックすれば、正しく解除されるように修正しました。バツボタンやドロワーのバックドロップ（背景部分）をクリックした場合は、ハイライトを維持します。

### 6. デザインの一貫性向上
- **計算式カラーの同期**: サイドドロワー内の計算式ハイライト（`FormulaHighlighter`）の配色を、バッジのポップアップで使われている配色と完全に一致させました。また、行番号表示付きのテーブルレイアウトを採用し、視認性を向上させました。

### 7. 依存関係の定義修正とナビゲーション改善
- **依存関係の向きを正視化**: 「上流（Upstream）＝参照元（この項目が参照している項目）」および「下流（Downstream）＝参照先（この項目を参照している項目）」というデータフローに基づいた正しい定義とラベルに修正しました。
- **ドリルダウン時の「戻る」機能**: サイドドロワー内で依存関係を辿った際、直前の項目に簡単に戻れるよう、ヘッダーに「戻る」ボタンを追加しました。これにより、複雑な計算式の連鎖をスムーズに探索できるようになりました。

### 8. 安定性の向上とバグ修正
- **インポート漏れの修正**: `App.tsx` および `SideDrawer.tsx` において、`useEffect` や `useState` などの React フックのインポートが漏れていたため、ファイルアップロード後に画面が真っ白になる問題を修正しました。
